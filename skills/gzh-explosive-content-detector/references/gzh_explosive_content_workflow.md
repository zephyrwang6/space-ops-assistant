## 核心功能

为公众号文章创作打造的专业数据洞察工具。全网持续收录每日超过1000条10w+阅读的文章，收录依据：低粉爆款、10w+阅读、原创文章，助力号主获取数据参考

## 前置准备

- 依赖：`requests>=2.28.0`

## 重要数据说明

- **数据时间差说明**：文章互动数据截止为入库时间，不是实时数据，展示数据仅为入库快照，互动量可能持续增长。
- 数据库仅包含昨天至30天前的数据
- 默认时间范围为最近7天（today-7天）
- 数据不足自动扩展范围，并提示用户

## ⚠️ 核心执行规则（必须遵守）

1. **泛化词必须先询问再查询**：当识别到泛化词时，**绝对禁止直接调用脚本**，必须先输出细分词推荐并等待用户选择后再执行查询
2. **正确执行顺序**：关键词提取 → 判断是否泛化词 → 是泛化词则询问用户 → 用户回复后再调用脚本
3. **强制等待规则**：输出细分词推荐后，必须停止执行，等待用户下一轮对话回复「拓展」或「不拓展」，**不得在同一次对话中继续执行任何脚本调用**

**常见泛化词**：
泛词：抽象层级高、覆盖范围广的概括性词汇，无具体场景/属性修饰，行业分类等，可包含多个子类。特征：①语义上为上位概念（如"职场"包含"工作/打工人"；"情感"包含"恋爱/婚姻"；如AI）；②上下文中常搭配"领域""类型"等概括词（如"职场领域""情感类型"）。
**常见具体词**：
具体词：抽象层级低、指向明确的实例化词汇，含具体场景/属性修饰，属于某泛词的直接子类。特征：①语义上为下位概念（如"职场沟通"是"职场技能"子类；"恋爱技巧"是"情感"子类）；②词语结构多含修饰成分（如"婆媳"→"婆媳关系"；"亲子"→"亲子教育"）；

## 操作步骤

### 功能一：爆款文章探测

**核心功能**：输入关键词，获取最匹配的公众号爆款内容

#### 步骤1：用户意图理解（查询脚本前）

**⚠️ 核心规则：优先提取用户描述中的细分方向词，而非泛化的大类词**

1. **判断用户是否提到赛道关键词**：
   - **无赛道关键词**（如"最近的爆款文章有哪些"、"最近有什么热门内容"、"看看爆款数据"）→ 直接调用脚本，关键词传空字符串 `""`，查询全站热门
   - **有赛道关键词** → 继续提取和判断

2. **提取精确搜索关键词**（仅当用户提到赛道时执行）：
   - **分析用户描述**：从用户自我介绍或需求描述中提取明确的细分领域词
   - 示例分析：
     - 用户输入："我是一个情感类公众号博主，平时会发恋爱技巧、婚姻经营、婆媳关系等相关内容，帮我找情感领域热门话题"
     - 分析结果：用户提到的细分方向 = 恋爱技巧、婚姻经营、婆媳关系
     - 将前文场景和"情感"相关，得到细分词 =情感、恋爱技巧、婚姻经营、婆媳关系
     - 搜索关键词：情感、恋爱技巧、婚姻经营、婆媳关系
     - ❌ 错误做法：只提取泛化词「情感」去搜索

3. **关键词类型判断**（仅当提取到关键词时执行）：
   - **细分词/垂直赛道**（含具体场景/属性修饰的词，如"职场沟通技巧"、"亲子教育"、"婆媳关系"）→ 直接搜索，无需拓展询问
   - **泛化词/分类**（纯大类词，如"职场"、"情感"、"育儿"，无任何修饰）→ 执行拓展策略（步骤2）
   - **判断原则**：有修饰词（场景/人群/风格/意图）= 细分词，直接搜索；无修饰词 = 泛化词，需要拓展

#### 步骤2：泛化词拓展策略

1. **泛化词处理流程（⚠️ 必须等待用户明确回复后再调用脚本！）**：
   1. **第一步：生成细分词**（禁止调用脚本搜索数据）
      - 拓展词生成原则：
        - **词的大小适中**：词语不要加组合，避免过细（如"职场新人穿搭"太细，查不到数据），
        - **必须覆盖不同场景**：趋势词、人群词、场景词、意图词各2-3个

      - 输出示例：

      ```

      ```

      我识别到「职场」是较大的分类，已查询近期热门趋势，推荐以下细分方向：
      工作、打工人、职场穿搭、职场干货、成长、冷门职业、向上管理、职场焦虑、升职、财务自由
      回复「拓展」将同时搜索这10个词，回复「不拓展」将继续搜索「职场」

      ```

      ```

   2. **第二步：等待用户回复**
      - ❌ **禁止**：用户未回复时调用脚本
      - ✅ **正确**：只等待用户明确回复「拓展」或「不拓展」后再执行
   3. **第三步：根据用户明确回复执行**
      - 用户回复「拓展」 → 调用脚本搜索10个细分词
      - 用户回复「不拓展」或「继续」 → 调用脚本搜索原关键词
      - 用户未回复或回复其他内容 → 识别对应意图

#### 步骤 3：时间范围与数据查询

**时间范围**：

- 数据库只包含昨天至30天前的数据
- **"最近"的默认定义**：最近7天（startDate = 今天 - 7天）
- **日期计算**（将用户表达转换为 startDate）：
  - 今天：直接用昨天日期，startDate = 昨天
  - 最近/近7天：startDate = 今天 - 7天
  - 近N天：startDate = 今天 - N天
  - 示例：用户说"近15天" → startDate = 今天 - 15天

**数据不足时的自动调整（⚠️ 优先扩展时间！）**：

- **处理原则**：数据不足时，先扩展时间范围，拓展时间后数据仍然不足，可以拓展关键词
- **调整顺序**：按以下顺序自动扩展时间范围
  1. 近1天 → 近3天
  2. 近3天 → 近7天
  3. 近7天 → 近30天
- **告知用户**：自动调整时告知用户："该关键词近X天数据较少，已自动扩展时间范围至近Y天"
- **禁止行为**：❌ 不可因为数据不足就更换关键词、推荐其他词或触发泛化词拓展流程

**超出范围或未更新数据的道歉说明**：

- 用户说"今天/今日"时：回答"非常抱歉，今天的数据暂未更新，已为您展示最近可用的数据"
- 用户要求的时间超出30天时：回答"非常抱歉，当前仅支持最近30天的数据，已为您展示最接近的数据"

**执行命令**：

- 有赛道关键词：`python scripts/fetch_gzh_trends.py --keyword <关键词> --start-date <日期>`
- 无赛道关键词（查询全站热门）：`python scripts/fetch_gzh_trends.py --keyword "" --start-date <日期>`
- 多个关键词用逗号分隔：`python scripts/fetch_gzh_trends.py --keyword "职场沟通,恋爱技巧,亲子教育" --start-date <日期>`
- source 参数固定传值：接口会自动传入 source=公众号爆款文章洞察

**输出文件**：

- 筛选后推荐数据：`{keyword}_爆款数据.html`

##### 4.1 意图识别（必须先执行）

**在筛选前，必须先判断用户意图类型**：

| 意图类型       | 特征                 | 示例                                           |
| -------------- | -------------------- | ---------------------------------------------- |
| **泛浏览型**   | 无明确目标，想看热门 | "有什么好看的"、"推荐一下"、"看看爆款"         |
| **明确需求型** | 有具体目标/场景      | "找职场沟通技巧"、"适合情感号"、"亲子教育内容" |
| **数据需求型** | 关注数据表现         | "高赞内容"、"高分享"、"互动最多的"             |

**输出意图判断**（简短说明，用于后续筛选依据）：

```
意图判断：用户为【明确需求型】，目标人群"职场新人"，核心需求"团队融入技巧"
```

##### 4.2 综合评分维度

脚本已计算数据表现分数（0-100分），智能体需结合意图进行最终筛选：

| 维度           | 权重 | 说明                                                   |
| -------------- | ---- | ------------------------------------------------------ |
| **意图匹配度** | 最高 | 是否直接回应用户问题/目标人群/核心需求                 |
| **内容相关性** | 高   | 标题关键词、正文主题、标签精准度                       |
| **数据表现**   | 中   | 点赞/评论/分享/阅读数/互动总数（脚本已提供 dataScore） |
